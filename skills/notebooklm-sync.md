# Skill: NotebookLM Sync

## Purpose

Sync a local ABW package to NotebookLM through the `nlm` CLI after `/abw-pack` has produced a package under `notebooks/packages/<package_id>`.

## Safety Rules

1. Dry-run is mandatory before execution.
2. Do not sync packages with `qa_status: fail`.
3. Do not sync packages with `qa_status: needs_review` unless the user explicitly requests an override.
4. Do not delete NotebookLM sources in Phase 4.
5. Do not create or select a notebook by inference. Use only a notebook ID or alias explicitly provided by the user.
6. If authentication or network fails, stop and report the exact failure.

## Script Contract

Use:

```bash
python ~/.gemini/antigravity/scripts/abw_sync.py --package-dir <path> --notebook-id <id>
```

Dry-run returns JSON and writes:

```text
<package-dir>/sync_report.json
```

Exit codes:

```text
0 = dry-run ok or sync completed
1 = runtime or nlm execution error
2 = blocked because package needs_review
3 = blocked because package failed QA or exceeds source limit
```

## Operator Report

After running the script, summarize in Vietnamese:

```text
Package:
QA status:
Notebook target:
Sources to upload:
Mode: dry-run | execute
Decision: blocked | ready for explicit execute | synced
```
