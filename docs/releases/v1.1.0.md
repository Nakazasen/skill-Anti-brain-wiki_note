# ABW v1.1.0 Release Notes

## Overview
Release v1.1.0 focuses on stabilizing the trust-based retrieval architecture and hardening the multi-workspace routing logic. This version introduces a robust "route-then-ask" pipeline that ensures queries are directed to the most appropriate context while maintaining strict guards against domain ambiguity.

## Key Changes

### 1. Trust/Retrieval Architecture Hotfix
- Implemented enhanced trust score calculation for grounded wiki notes.
- Improved retrieval status normalization to distinguish between `exact_match`, `fuzzy_match`, and `no_match`.
- Fixed character encoding issues in API response payloads.

### 2. Workspace Registry & Routing Hardening
- Introduced a dynamic workspace registry system using `.brain/workspaces.json`.
- Implemented `route_query` logic to assess workspace purpose using semantic and token-based profiles.
- Added a "Generic Term Filter" to prevent broad queries (e.g., "hệ thống", "system") from forcing incorrect routing decisions.

### 3. Multi-Workspace Route-then-Ask Pipeline
- Automated the transition from global `auto` asking to specific workspace grounding.
- The system now performs a pre-flight route check before executing the `ask` logic, ensuring the retrieval engine operates on the best available context.

### 4. Domain Isolation & Ambiguity Guards
- Implemented explicit markers for `industrial_documentation` and `web_novel_platform`.
- Added logic to detect and flag cross-domain ambiguity (e.g., mixing industrial and fiction terms), defaulting to `ambiguous` instead of forcing a low-confidence match.
