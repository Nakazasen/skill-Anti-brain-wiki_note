## Goal

Define a repeatable retrieval scoring harness before adding new parsers. The harness should measure whether ABW answers are grounded, bounded, and honest about missing evidence across verified, recoverable, partial, and unsupported workspaces.

Non-goals:

- Do not add a DOCX parser.
- Do not expand source ingestion behavior.
- Do not weaken fail-close answers.
- Do not treat missing evidence as success.

## Fixed Eval Question Format

Each eval question should be stored as structured data so the same question can run repeatedly across releases.

Required fields:

```yaml
id: MOM-TARGET-001
workspace_id: A
workspace_path: D:\Sandbox\MOM_WMS_QLLSSX
mode: targeted
question: What documents describe AGV communication?
intent: Find specific evidence about AGV communication in local project sources.
expected_behavior: grounded_answer
expected_source_coverage:
  required_any:
    - agv
    - communication
  preferred_extensions:
    - pdf
    - txt
    - html
minimum_grounding_score: 3
maximum_missing_evidence_score: 1
allow_bounded_partial: false
must_log_gap_if_insufficient: true
overclaim_checks:
  - must_not_claim_full_corpus_review_without evidence
  - must_not_invent_source_titles
```

Recommended question modes:

- `targeted`: asks for a narrow fact, source, procedure, or definition.
- `broad_summary`: asks for a bounded summary over retrieved evidence.
- `negative_control`: asks for likely missing information and expects a gap, not a fabricated answer.
- `unsupported_format`: asks about known unsupported raw material and expects clear unsupported-corpus handling.

## Expected Source Coverage Format

Expected source coverage is not an exact-answer script. It defines what evidence should be present for an answer to count as grounded.

Coverage fields:

```yaml
expected_source_coverage:
  required_any:
    - keyword or source family that should appear in at least one cited chunk
  required_all: []
  preferred_files: []
  preferred_extensions:
    - pdf
    - txt
  forbidden_sources:
    - unrelated generated draft
  minimum_distinct_sources: 1
  minimum_distinct_chunks: 1
```

Rules:

- `required_any` prevents brittle exact-file coupling while still checking topical grounding.
- `required_all` should be used sparingly for facts that must co-occur.
- `preferred_files` should be used only when a known gold source exists.
- `forbidden_sources` catches drift into irrelevant drafts or unrelated project files.
- Unsupported workspaces may have `minimum_distinct_sources: 0` when the expected behavior is a missing-evidence response.

## Answer Grounding Score

Grounding is scored from `0` to `4`.

| Score | Meaning |
| --- | --- |
| 4 | Answer is directly supported by cited local evidence, cites multiple relevant chunks when available, and preserves source scope |
| 3 | Answer is supported by at least one relevant local source and does not overclaim |
| 2 | Answer is partially supported but misses expected source coverage or has weak citation linkage |
| 1 | Answer mentions plausible project terms but provides little verifiable source support |
| 0 | Answer is ungrounded, fabricated, or sourced from unrelated evidence |

Targeted questions should usually require `3` or higher. Broad summaries may allow `2` only when the answer explicitly states partial coverage and logs gaps.

## Missing-Evidence Score

Missing-evidence handling is scored from `0` to `3`.

| Score | Meaning |
| --- | --- |
| 3 | Correctly identifies insufficient evidence, states what is missing, and logs or reports a gap |
| 2 | States insufficient evidence but gives incomplete gap detail |
| 1 | Hints at uncertainty but still answers beyond available evidence |
| 0 | Hides missing evidence or fabricates a complete answer |

For negative-control and unsupported-format questions, this score is the primary pass signal. A missing-evidence response can pass even when grounding is low, provided it does not overclaim.

## Hallucination and Overclaim Checks

Each eval result should run explicit checks before it can pass.

Failing checks:

- Claims full-corpus reasoning when only bounded retrieval was used.
- Invents source titles, file names, counts, dates, or command results.
- Treats unsupported DOCX content as parsed evidence.
- Converts a recoverable or unsupported corpus into a confident project summary.
- Omits a knowledge gap when required evidence is missing.
- Presents old ABW state, runtime drift, or unsupported corpus warnings as solved without evidence.

Warning checks:

- Uses vague source language such as "the documents say" without naming retrieved evidence.
- Gives a broad summary with no explicit scope boundary.
- Relies on only one weak source for a question that expects multiple sources.

## Bounded Summary Scoring

Broad summaries should be scored separately from targeted retrieval because a safe partial answer is often better than a confident complete answer.

Bounded summary score from `0` to `4`:

| Score | Meaning |
| --- | --- |
| 4 | Summarizes retrieved evidence, names scope boundaries, identifies gaps, and avoids full-corpus claims |
| 3 | Provides a useful partial summary with clear limitations |
| 2 | Provides a limited summary but weakly explains evidence boundaries |
| 1 | Summary is mostly generic or weakly grounded |
| 0 | Summary overclaims, fabricates coverage, or ignores missing evidence |

Minimum acceptance:

- Verified workspace broad summaries should score at least `3`.
- Recoverable partial workspaces should score at least `2` plus a missing-evidence score of at least `2`.
- Unsupported corpus workspaces should not be required to summarize unsupported content; they should pass by reporting the unsupported condition and missing evidence.

## Workspace-Level Eval Report

Each eval run should produce a workspace-level report that separates corpus readiness from answer quality.

Required report fields:

```yaml
workspace_id: A
workspace_path: D:\Sandbox\MOM_WMS_QLLSSX
abw_version: 0.5.0
baseline_version: 0.4.1
workspace_state: verified
corpus_readiness: healthy_supported_corpus
question_count: 6
passed: 5
failed: 1
warnings: 2
scores:
  average_grounding: 3.2
  average_missing_evidence: 2.7
  average_bounded_summary: 3.0
failures:
  - question_id: MOM-SUMMARY-002
    reason: broad summary did not name retrieval boundary
gaps_logged:
  - question_id: MOM-NEGATIVE-001
    gap: missing vendor-specific AGV retry policy
unsupported_sources:
  docx: 0
  xls: 1
  xlsm: 1
runtime_notes:
  release_match: unverified_wheel_release
  old_abw_state: true
```

Pass criteria at workspace level:

- No hallucination or hard overclaim failures.
- All targeted questions meet their minimum grounding score or correctly log missing evidence.
- Broad summary questions meet bounded summary thresholds for the workspace state.
- Unsupported corpus questions do not claim unsupported files were parsed.
- Report includes corpus readiness, unsupported source counts, and gap outcomes.

## Recommended v0.5.0 Priority

1. Implement eval data loading and result schema.
2. Add a read-only eval command that runs fixed questions and writes reports.
3. Score grounding, missing evidence, bounded summaries, and overclaim checks.
4. Add fixture-based tests for verified, recoverable partial, empty, and unsupported corpus states.

Explicitly defer:

- DOCX parser support.
- OCR or vision provider fallback.
- Automatic corpus cleanup or migration.
- LLM-based grading beyond deterministic checks until the report schema is stable.
