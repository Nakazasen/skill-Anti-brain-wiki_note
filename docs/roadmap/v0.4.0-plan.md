# ABW v0.4.0 Roadmap Plan

This document proposes the next major direction after the `v0.3.13` stable baseline. It is planning-only and does not describe shipped `v0.4.0` behavior.

## Priority Order

1. Multi-project deployment hardening
2. Retrieval quality evaluation harness
3. Corpus cleanup/migration tooling

## Candidate Themes

### 1. Multi-project Deployment Hardening

Highest priority because v0.3.13 is validated mainly on one real MOM workspace.

Focus areas:

- Define a small validation matrix across multiple workspace shapes.
- Verify stale-install and runtime-source reporting in each workspace.
- Confirm package/runtime version reporting remains understandable outside the source repository.
- Preserve existing workspace data and avoid any automatic trusted wiki promotion.

### 2. Retrieval Quality Evaluation Harness

Second priority because broad summaries and targeted retrieval need repeatable scoring before retrieval behavior is expanded.

Focus areas:

- Add fixed evaluation questions for targeted retrieval and broad summary behavior.
- Record expected source coverage for each question.
- Check knowledge gap logging when evidence is missing.
- Define acceptance criteria for bounded partial summaries so they remain honest about scope limits.

### 3. Corpus Cleanup/Migration Tooling

Third priority because MOM retained historical conflict reports and orphan drafts by design.

Focus areas:

- Start with audit and dry-run reports only.
- Separate active drafts, orphan drafts, historical conflict reports, and promoted wiki material.
- Require explicit review before deletion, migration, or archival.
- Keep auditability as the default behavior.

## Explicitly Deferred

- OCR/vision provider fallback: useful, but it should follow retrieval and evaluation baselines so OCR improvements can be measured.
- CI release pipeline: valuable, but less urgent than proving multi-project behavior and retrieval quality.

## Public APIs and Interfaces

No public API, CLI, schema, or version changes are planned by this roadmap note.

The `v0.4.0` items above are candidate implementation themes only. They must not be treated as shipped behavior until implemented, tested, and released separately.
