# Roadmap: v0.7.3 Recovery Advisor

## Overview
The **Recovery Advisor** (`abw recover-plan`) is a read-only diagnostic layer designed to transform raw corpus gaps and evaluation warnings into a structured, step-by-step resolution roadmap. It provides users with clear commands and estimated quality gains, bridging the gap between "knowing what's wrong" and "fixing it safely."

## Status: Proposed (v0.7.3)
Target Release: v0.7.3

## Core Features
1. **Impact-Based Ranking**: Prioritizes gaps that block the most evidence (e.g., `format_block` > `stale_draft_noise`).
2. **Actionable Command Generation**: Outputs exact CLI commands for the user to copy-paste.
3. **Quality Gain Projection**: Estimates how much the `eval` grounding score will improve after each step.
4. **Verification Loop**: Suggests a re-test sequence (`eval` -> `gaps`) to validate fixes.

## Proposed Spec: `abw recover-plan`

### Output Structure
```text
ABW Recovery Plan
-----------------
Workspace: <path>
Health State: <doctor_state>
Corpus Health: <inspect_summary>

Top 3 Recovery Steps:

1. [CRITICAL] Fix Unsupported Formats
   - Action: Convert 817 .docx files to .pdf or .txt
   - Command: abw ingest raw/*.pdf (after conversion)
   - Est. Gain: +4.0 Grounding Score
   - Impact: Blocks 100% of raw evidence

2. [HIGH] Reduce Draft Noise
   - Action: Review or archive 153 stale drafts
   - Command: abw review --archive-stale
   - Est. Gain: +1.5 Retrieval Precision
   - Impact: Pollutes top-K ranking

3. [MEDIUM] Bridge Wiki Gaps
   - Action: Create wiki notes for 5 target topics
   - Command: abw save --wiki "Topic Summary"
   - Est. Gain: +2.0 Evidence Anchoring
   - Impact: No grounded anchors for raw sources

Verification Sequence:
1. abw ingest ...
2. abw eval
3. abw gaps
```

## Implementation Plan

### Phase 1: Ranker Engine
- Implement a weight-based ranker for `GapType`.
- `format_block`: Weight 1.0 (Critical)
- `missing_wiki_coverage`: Weight 0.8 (High)
- `identity_gap`: Weight 0.7 (High)
- `stale_draft_noise`: Weight 0.4 (Medium)

### Phase 2: Action Mapping
- Map `GapType` to specific `abw` commands.
- `stale_draft_noise` -> `abw review`
- `missing_wiki_coverage` -> `abw save --wiki`
- `format_block` -> Manual conversion tip

### Phase 3: Score Projection
- Static estimates based on historical eval data.
- Grounding score usually jumps 2-3 points when wiki notes are added.
- Failure rate drops ~80% when formats are supported.

## Success Criteria
- [ ] Command is read-only (no side effects).
- [ ] Plans are workspace-specific.
- [ ] Accuracy of action recommendations > 90%.
