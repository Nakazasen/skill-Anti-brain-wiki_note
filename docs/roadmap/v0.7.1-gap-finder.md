# ABW v0.7.1 Roadmap: Retrieval Gap Finder
> Version: 1.0.0

## Purpose
The **Retrieval Gap Finder** is designed to identify exactly *why* retrieval fails for specific topics or questions by correlating evaluation failures with corpus intelligence diagnostics. It bridges the gap between "what happened" (eval reports) and "what is available" (inspect reports).

## Inputs
The Gap Finder will analyze the following read-only artifacts:
- **Eval Reports**: Identified failures and grounding scores.
- **Inspect Reports**: File counts, extensions, and draft states.
- **Doctor State**: System health, provider status, and config validity.
- **Corpus Counts**: Raw vs. Processed distribution.

## Diagnostic Output Examples
- **Identity Gaps**: "AGV topic lacks wiki notes; identity found in path but missing in indexed content."
- **Indexing Gaps**: "MP2027 spreadsheets (.xlsx) exist but processed state is stale or missing."
- **Format Blocks**: "Website corpus blocked by 817 unsupported DOCX files."
- **Noise Interference**: "Too many stale drafts (153) are likely polluting retrieval ranking."

## Suggested Next Actions
- `abw ingest raw/`: Recommended when raw files exist but processed count is low.
- `abw save --wiki`: Recommended when a topic matches workspace identity but has no wiki coverage.
- `convert docx to pdf/txt`: Recommended when unsupported formats dominate.
- `abw repair --stale-drafts`: Recommended when high draft counts impact precision.
- `abw rebuild processed`: Recommended when doctor flags index drift.

## System Invariants
- **Read-Only**: The Gap Finder only reports findings. It never modifies the corpus, config, or code.
- **Fail-Close**: Diagnostics must acknowledge when evidence is missing or ambiguous.

## Milestone Plan

### Milestone 1: Gap Detection (M1)
- Implement correlation logic between `EvalResult` failures and `build_inspect_report` stats.
- Identify "Missing Knowledge" vs. "Blocked Format" vs. "Stale Index" categories.

### Milestone 2: Impact Ranking (M2)
- Rank gaps by failure frequency.
- Prioritize actions that resolve the highest number of failed eval cases.

### Milestone 3: Export & Actionability (M3)
- Generate a `gap_report.json` artifact.
- Integrate findings into `abw inspect` as "Critical Retrieval Gaps".
