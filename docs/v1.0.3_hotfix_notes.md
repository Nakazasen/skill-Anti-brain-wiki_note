# ABW Admin v1.0.3 Hotfix Notes

## Feedback Triage

| Severity | Feedback | Count | Likely duplicate | Disposition |
| --- | --- | ---: | --- | --- |
| Critical | Crash on startup | 1 | Startup confusion / path setup failures may present as startup crashes | Fixed defensively in UI startup and connection guidance |
| High | Menu too small | 1 | Small fonts / cramped navigation controls | Fixed with larger app text, wider side panels, and taller buttons |
| Medium | Add dark mode | 1 | None | Deferred; feature request, not a hotfix |
| Low | None observed | 0 | None | No action |

## Duplicate Analysis

- `Crash on startup` is likely related to the broader startup confusion/path issues class because startup, API launch, and first workspace selection share the same first-run flow.
- `Menu too small` is a standalone legibility issue, but it overlaps with general readability complaints such as small fonts.
- `Add dark mode` is not a duplicate of the bug/usability reports and is out of scope for this hotfix.

## Top Usability Pain Points

1. First launch can fail or look failed without clear guidance when the API is unreachable.
2. Navigation and result text were too small for comfortable pilot use.
3. Workspace path defaults and errors were not explicit enough for users launching from different directories.

## Fixes Included

- Bumped the desktop app version to `1.0.3`.
- Increased default UI readability with larger fonts, wider panels, taller buttons, and taller feedback table rows.
- Defaulted the workspace field to the current absolute working directory instead of `.` when no previous workspace exists.
- Added workspace validation before workspace-scoped actions.
- Improved API timeout and connection error messages so users get actionable next steps instead of generic failures.
- Made feedback dashboard export/review actions create the feedback directory when needed.

## Deferred

- Dark mode is deferred because it is a feature expansion, not a safe hotfix.
- API threading/background execution is deferred because it has higher regression risk than the pilot feedback justifies for v1.0.3.
